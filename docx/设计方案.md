# 项目设计方案：实训表现数字化分析平台

## 1. 项目概述
本方案旨在为《服务器技术应用》项目一提供数据可视化支持。通过解析 Excel 成绩表，系统将 8 个测试点自动归类为 5 大核心技术维度，并以 Web 雷达图仪表盘的形式展示每个学生的薄弱项。

## 2. 技术架构 (SPA 模式)
* **前端框架**: 原生 HTML5 / CSS3 / JavaScript (ES6+)。
* **图表库**: [ECharts.js](https://echarts.apache.org/) (百度开源的高效可视化库)。
* **数据解析**: [SheetJS (XLSX.js)](https://sheetjs.com/) (用于浏览器端解析 Excel 文件)。
* **部署方式**: 纯静态页面，无需后端服务器，支持离线运行。

## 3. 数据模型设计
系统将对原始题目进行**百分制标准化**映射：

| 维度 (Dimension) | 关联题目 | 权重 | 考核技术点 |
| :--- | :--- | :--- | :--- |
| **磁盘存储管理** | (1), (2) | 20% | `parted` 分区、`/etc/fstab` 挂载 |
| **基础环境构建** | (3), (7) | 30% | YUM 本地源、IP 地址网络配置 |
| **安全合规配置** | (4), (8) | 20% | `firewalld` 防火墙策略 (DNS/HTTP) |
| **DNS 服务治理** | (5) | 10% | `bind` 域名解析记录与校验 |
| **Web 业务应用** | (6) | 20% | Apache 站点部署与连通性测试 |

## 4. 核心功能模块

### 4.1 智能数据导入
* **功能描述**: 支持拖拽或选择 Excel 文件上传。
* **智能映射**: 自动识别表头关键字（如：“姓名”、“第一题”、“(1)”）。
* **容错处理**: 若某项数据缺失，系统默认补 `0` 并标记。

### 4.2 仪表盘列表展示
* **栅格布局**: 页面采用响应式布局，每行展示 3-4 个同学的雷达图。
* **可视化效果**: 
    * 雷达图背景采用 5 层等高线。
    * 填充区域使用半透明主题色，边缘高亮显示。

### 4.3 阶梯式自动评价 (Logic)
系统根据每个维度的**标准化得分 ($S$)** 自动生成实时反馈：
* **$S \ge 90$**: 🌟 **熟练掌握** - 该项技术掌握非常扎实。
* **$60 \le S < 90$**: 👌 **基本达标** - 能够完成任务，但细节需优化。
* **$S < 60$**: ⚠️ **预警建议** - 
    * *存储维度*: 需加强分区持久化练习。
    * *网络维度*: 需熟悉 IP 配置与 YUM 环境。
    * *安全维度*: 需关注防火墙策略放通。

## 5. UI/UX 设计要点
* **配色方案**: 采用深色模式（适合信息安全专业酷炫感）或 简洁学术蓝。
* **交互交互**: 鼠标悬停在雷达图顶点时，浮窗显示该维度的具体得分。
* **搜索过滤**: 顶部增加搜索框，可根据“姓名”快速锁定特定学生的分析图表。

---

## 6. 开发计划 (Roadmap)
1.  **Phase 1**: 环境搭建，引入 ECharts 与 XLSX 库。
2.  **Phase 2**: 编写 JavaScript 解析逻辑，实现 Excel 转 JSON。
3.  **Phase 3**: 编写标准化算法，将得分映射至 5 个维度。
4.  **Phase 4**: 循环渲染全班雷达图及评价文本。
5.  **Phase 5**: 最终调试与样式优化。
